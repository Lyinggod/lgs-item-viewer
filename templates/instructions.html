Lyinggod's Item Enhancements

Instructions:

This module is optimized to be used with the Item Tags module and provides a variety of options that improve how items interact with the actors (PCs and NPCs) and the player, as described below. It also provides tweaks to the Furukai's Simple Crafting module by allowing the use of difficulty codes in the receipe names.

Species and Career Improvements

Items may be assigned by Species and Careers to actors. This is done via an additional red bar that can be found in Species under the Talents tab and on Careers under the Career Data tab. A list of assigned items will appear below this bar. Click on the Items (+) in the red bar to display the Item Assignment dialog.

Adding Items to a Species or Career

  • Items assigned to boxes in the Assignment Dialog will be automatically created on the actor (if they are normally allowed on an actor) when the career or species is dropped on the actor.
  • Boxes in the Assignment Dialog that include an or option will result in a dialog being shown when the Career/Species is dropped on an actor, prompting the user to select from item choices.
  • In the Assignment dialog, drop any item or the or button on one of the squares. The or button defines groupings based on the position of the or within a box. This includes talents and abilities.
  • Items can be moved between boxes.
  • Re-order items within a box by dragging and dropping items within the same box, in which case the item will move to the right most position in a box.
  • Right click to delete an item.
  • Click Save to store item positions.
  • Note: Native functionality for assigning Talents and Abilities should be used when possible.

Talent and Abilities on NPCs

This module will add talents and abilities if dropped on a NPC. These should be assigned using the default method of assigned these and not the previously described method.

Assigned Currency

An item with a name of "1d100+5 credits" or "500 credits" will assign a random or fixed number of currency to the actor (except minions). This item won't be created on the actor. Note: The word "credits" is required.

Catalog Viewer

When first installed, the module creates a compendium folder called “Catalog Viewer Compendiums”. Any compendiums of item type or actor type can be placed in the folder and will appear in the viewer. However, the viewer will only show actors that are vehicles, rivals, or minions.

When a table is created, click on the line to see the description of the item.

Organizing Compendiums

When placing items in compendiums, they must be placed in a folder as the folders are used by the viewer for navigation. Items that are not in folders may not appear correctly in the viewer. Folder names should reflect the general contents of the folder.

Groups Section

This section of the viewer contains a list of checkboxes of the item types that are present in the compendiums. Minions and Rivals will appear in a single item type "Special" (see below).

Category Tags Section

For each checkbox in Groups that is checked, a tag block of checkboxes will be created. The checkboxes are used to filter the visible items in their respective tables. The checboxes in the blue area are the folders the items were placed in. Below the blue area will be checkboxes with any tags that were defined in the Item Tags module.

Item Tables

Each type of item will appear on the left side of the viewer in its own table. This means that if there are 2 compendiums with "gear" type items in them, they will all appear under one table. Each table will have a header identifying its item type.

Purchasing Items
Items may be purchased by clicking on the shopping cart next to each item. Players will have the option to be their assigned character for the item being purchased. If purchasing a minion, rival or vehicle, players will have the option of having the purchase created on their character sheet as gear item or the in actor tab with the character's name as a prefix; ie if Dragmar buys a wagon it will appear as Dragmar's Wagon.

Alternately, the icon/text to the left of the shopping cart may be used to drag an item to the character sheet without purchase. This option will not work for minions, rivals, or vehicles.

Updating Items

Clicking on the item button (to the left of the shopping cart) will display the item sheet. This is the same as opening the item from the compendium. If an item's sheet is updated, the group checkbox must be unclicked and re-clicked to see the update.

When adding folders inside button compendiums or adding new compendiums to the button folders, the game will need to be refreshed.

Additional Configuration

When the module is activated the journal "Catalog Viewer Journal" is created. This journal is used to add supplemental information to the table header (based on item type) or folder based checkboxes in the Tag List section.

To add additional information to tables or categories (these are the same as the folder names), create a page with the name of the button (ie Equipment) with a page level of 1 (without page content). This must be followed immediately by a page with the name of the group (ie Gear) or folder/tag name (ie Laser Pistols), which may be of page level of 2 or 3. These last 2 types should have content. Group names will appear at the top of their table. Tags/folder names will appear with a "(+)" next to name and, when clicked, will display the contents below the tag. Tag/folder content only applies to the blue area.


Example
Equipment (level 1 page, this is the button name)
— Gear (level 2 page, this is optional.)
—— Laser Pistols (level 3 page, this is optional)
Vehicles (level 1 page, a different button)

Hiding Items

Specific items can be hidden from a player by giving it a tag with the text of "hide". Entire groups can be hidden from players by adding "(-)" to folder name. For the GM, hidden items/groups will be identified next to the item name.

Additional Configuration

The following may be configured under Configuration Settings.

Configuring Buttons

Up to item 3 buttons can be created. When a button is provided a name, the button will appear on refresh and a folder will created with the of "ButtonName Compendiums".

Images for Item identification

Items may be identified by the folder name with graphic by clicking on the Catalog Category Images button. In this section, enter a folder name and assign a image. This can be used to help identify items at a glance in the viewer. The default value can also be assigned to include options from Font Awesome.

Special Category Name

These are collectively minions and rivals. In some settings they may be purchased perhaps a golems or drones. This options allows for the renaming of item type "Special" to be thematically relativant.

The Catalog Viewer is accessible via buttons at the bottom of the Compendiums. By default, there is one button with the name of “Catalog Viewer”. 
In the compendium Lyinggods Macros is macro Catalog Viewer. It can be run from the folder, hot bar, or from the buttons at the bottom of the compendium.

The three buttons are associated with 3 folders which are created by the default; Character Compendiums folder, Equipment Compendiums folder, and Vehicle Compendiums folder. These can be turned off in Configure Settings. 

Updating Items

Items that are updated by opening the item via the Catalog Viewer interface will need to have their Group checkbox toggle to see changes in the the table.

Hiding Items from Players

Individual Items can be hidden from players by placing a “hide” tag (Item Tags module) on the item. To hide an entire folder from the players, add “(-)” to the name of the folder in the compendium. The GM will see an identifier on an item to show if an item is hidden or part of a hidden category.

Limitations

  • Deleting a career or species will not delete items assigned by this module.
  • This module uses references to items. Do not change the location of an item once the object is assigned to the species or career via the Item Assigner dialog.
